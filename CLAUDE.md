# Claude Code Notes — daily-briefing-podcast

## What This Repo Is

A personal podcast RSS feed hosted on GitHub Pages. Episodes are generated by Claude (audio via TTS), committed as static files, and served at `https://jacobbrooke95.github.io/daily-briefing-podcast/feed.xml`. Apple Podcasts and other clients pull from this feed.

There is no build step or CI pipeline. Push to `main` = live.

## Key Files

- `feed.xml` — The RSS feed. This is the critical file. All episode metadata lives here as hand-edited XML `<item>` blocks.
- `EPISODES.md` — Step-by-step publishing guide with templates for daily briefings, deep dives, and series chapters.
- `scripts/validate-feed.py` — Validates feed.xml (XML well-formedness, bare ampersands, required fields, duration format, file existence). **Run before every commit that touches feed.xml.**
- `scripts/update-feed-metadata.sh` — Updates `<lastBuildDate>` to match the newest episode's `<pubDate>`.

## Publishing Workflow

1. Place audio in `episodes/`, artwork in `artwork/`, transcript in `transcripts/`
2. Add `<item>` block to the top of feed.xml (see templates in EPISODES.md)
3. Run `./scripts/update-feed-metadata.sh`
4. Run `python3 scripts/validate-feed.py`
5. Commit and push to `main`

## Feed Validation

A pre-commit hook (`.git/hooks/pre-commit`) automatically runs `validate-feed.py` when `feed.xml` is staged. If validation fails, the commit is blocked.

To run manually:

```bash
python3 scripts/validate-feed.py
```

The validator checks:
- XML well-formedness (catches bare `&`, unclosed tags)
- Bare ampersands outside `<![CDATA[` blocks
- Required fields on every `<item>` (title, guid, pubDate, enclosure, duration, episode number, episode type)
- `itunes:duration` format (must be `MM:SS` or `HH:MM:SS`, not raw seconds)
- `lastBuildDate` freshness
- Enclosure audio files exist on disk

## Common Pitfalls

- **Bare `&` in XML** — Use `&amp;` everywhere except inside `<![CDATA[...]]>` blocks. A single bare `&` makes the entire feed unparseable and Apple Podcasts will reject all episodes. This has caused outages before (commits `a929de2`, `ca1075b`).
- **`<content:encoded>` and `<description>`** — Wrap HTML content in `<![CDATA[...]]>` to avoid escaping issues. Plain-text `<description>` and `<itunes:summary>` fields must have all special characters escaped.
- **Duration format** — Use `MM:SS` (e.g., `9:50`) or `HH:MM:SS`. Do not use raw seconds (`1036`) or leading-zero hour (`0:14:07`).
- **Episode numbers** — Must be sequential integers. Check the current highest with `grep -m1 '<itunes:episode>' feed.xml`.
- **File sizes** — `<enclosure length="">` must be exact byte count. Use `wc -c < episodes/FILE.mp3`.
- **GUIDs** — Never reuse or change a `<guid>` after publishing. Apple Podcasts uses these as the canonical episode identity.
- **Artwork caching** — Apple caches artwork aggressively. If updated artwork doesn't appear, change the filename.

## Timezone

All `<pubDate>` values use `-0500` (CDT). Be consistent.
